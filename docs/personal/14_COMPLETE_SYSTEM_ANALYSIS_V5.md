## 2. Recent Critical Upgrades (March 2026 - v4.3/v5.0)

### A. The 64-Column Feature Store Upgrade
The biggest bottleneck to creating an accurate XGBoost ML model was insufficient data. 
- **The Upgrade:** The `signals` table in Supabase was upgraded from 15 basic indicators to full **64-column** telemetry. It now precisely extracts every variable (Stochastic, CCI, MFI, ADX, options flow parameters) generated by the Python Engine.
- **The Outcome:** The database natively constructs the exact **57-numeric features** the ML training script expects (`train_model.py`) via the SQL View `ml_training_export`. No messy CSV formatting required.

### B. The "Survivorship Bias" Fix
Previously, the n8n logic recorded signals into Supabase *after* checking if they resulted in a trade. This caused 90% of the market ticks (WAIT, SIDEWAYS) to be discarded. 
- **The Fix:** The `Log Signal to Supabase` node was repositioned to the *start* of the flow. 
- **The Outcome:** The database now collects negative and neutral states (`0=BUY CE`, `1=BUY PE`, `2=WAIT`). The XGBoost AI now has access to the negative training data required to learn what "not to do."

### C. React JSONB Parsing Interceptor
During the telemetry expansion, complex objects like Options Flow (GEX, IV Skew) were pushed to Supabase as JSONB strings. This initially broke the React dashboard (causing `NaN` errors).
- **The Fix:** A defensive telemetry interceptor `parseJSONNum` was built directly into `src/services/supabaseApi.ts`.
- **The Outcome:** The UI instantly parses and mounts double-encoded JSON payloads transparently, displaying intricate data like absolute Gamma Exposure natively in the dashboard.
