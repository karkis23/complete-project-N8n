## Phase 2: Technical Hardening — Making the System Bulletproof
**Timeline: Parallel to Phase 1**

While data is being collected, we identified three "blind spots" in the architecture that should be fixed before scaling:

### Blind Spot A: No WebSocket (HTTP Polling is slow)
The React dashboard currently refreshes every 30 seconds. If the internet drops, the UI silently stops updating. Fix: Upgrade to a WebSocket connection so the dashboard updates in real-time and shows an "offline" banner if internet drops.

### Blind Spot B: Silent Failover (No Alert When AI Falls Back to Rules)
If the AI model crashes and falls back to the rules engine, it does it silently. The trader doesn't know. Fix: Send a Discord or Telegram webhook alert if the engine switches from AI_ENSEMBLE to RULES_FALLBACK unexpectedly.

### Blind Spot C: Generic Error Messages
If Python fails internally, it sends a generic "500 Error" to the dashboard. Fix: Build custom exception routing in FastAPI to send human-readable error messages (e.g., "Missing Angel One API token").

## Phase 3: AI Training — Awakening the Machine 🟡 NEXT
**Timeline: April to May 2026**

**Step 1:** Run `api/scripts/oracle_labeler_v2.py` to label all signal rows using OHLC candle High/Low first-hit detection.
**Step 2:** Validate label distribution (≈ 37% CE, 26% PE, 36% WAIT based on simulation).
**Step 3:** Run `python api/scripts/train_model.py` with labeled data from `ml_training_export` view.
**Step 4:** XGBoost trains with GPU acceleration and generates `signal_xgb_v1.pkl`.
**Step 5:** Restart the Python server. It auto-detects the model. AI_ENSEMBLE mode activates.

From this point forward, signals are generated by machine learning probability, not hardcoded rules. The AI outputs confidence scores (e.g., "84.5% BUY CE") instead of binary yes/no rules.

## Phase 4: Continuous Retraining — Keeping the AI Sharp
**Timeline: Ongoing after Phase 3**

The market constantly evolves. What worked in May might not work in October. To keep the model sharp, we retrain it periodically.

**The "Weekend Retrain" routine:**
1. Every Saturday, download the latest Google Sheet CSV (which keeps growing weekly)
2. Run the Labeler on the new rows
3. Run `train_model.py` with the combined dataset
4. A new, smarter `signal_xgb_v1.pkl` is generated in ~60 seconds
5. Restart the server on Monday morning

No changes to n8n, no changes to the dashboard. The system seamlessly upgrades.

## Phase 5: The v5.0 Vision — LSTM Deep Learning
**Timeline: Late 2026**

XGBoost analyzes a single 5-minute candle snapshot. It is like looking at a photograph.

LSTM (Long Short-Term Memory) Neural Networks analyze the SEQUENCE of time. It is like watching a movie. The AI can look at the last 20 candles in order and understand the rhythm of institutional buying and selling.

The v4.0 architecture is specifically designed to support this upgrade without changing any API endpoints or n8n workflows. We simply swap the model behind the scenes.

Capabilities unlocked by LSTM:
- Detect Wyckoff Accumulation patterns (2-hour institutional buildup before a breakout)
- Sense "momentum fatigue" (a strong trend that is quietly losing steam)
- Spot late-day institutional distribution before it shows on any indicator
